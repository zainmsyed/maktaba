### Finding 3
- Severity: medium
- Category: bug
- Summary: Document deletion still uses raw stored file paths, so local-dev documents stored with Docker-style `/data/...` paths are soft-deleted but their PDF/cover files may remain on disk.
- Evidence: `backend/app/main.py` added `resolve_document_file_path()` and uses it for streaming/highlight extraction, but `delete_document` still removes files via `Path(doc.file_path)` and `Path(doc.cover_path)`. Existing documents in local dev can store paths like `/data/pdfs/...`, while the actual host files live under `DATA_DIR/pdfs/...`; the raw `Path('/data/...')` check will fail and silently skip deletion.
- Recommendation: Reuse `resolve_document_file_path()` in `delete_document` for `doc.file_path` and cover deletion, or add an equivalent resolver for cover paths. Add a regression test that stores a `/data/pdfs/...` path while `DATA_DIR` points at a temp host directory and verifies physical deletion.
